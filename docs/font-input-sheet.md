# 폰트 지정용 단일 입력 시트

이 문서는 현재 사이트의 실사용 화면에서 폰트 지정이 필요한 텍스트 인터페이스를 한 번에 모아둔 입력 시트입니다. 각 줄은 실제 화면 요소 하나를 가리키며, 사용자는 마지막 칸에 원하는 폰트만 채워 넣으면 됩니다. 이후 구현 단계에서는 이 항목명을 기준으로 전역 폰트 토큰이나 셀렉터 매핑으로 옮기면 됩니다.

기준 소스는 현재 활성 레이아웃과 `static/css/site.css` 입니다. 검색 화면의 Pagefind UI 는 포함했고, 댓글 영역은 사이트가 직접 그리는 패널 제목과 안내 문구만 포함했습니다. commentbox 위젯 내부 텍스트, 소셜 플랫폼 임베드 내부 UI, 장식용 아이콘과 마크는 폰트 지정 대상에서 제외했습니다.

## 전역 인터페이스

사이트 어디에서나 반복되는 텍스트부터 먼저 채우면 전체 분위기를 가장 빠르게 잡을 수 있습니다. 이 구간은 헤더, 내비게이션, 푸터, 접근성 링크처럼 전역 공통 요소를 다룹니다.

| 화면 요소 | 설명 | 사용자가 넣을 폰트 |
| --- | --- | --- |
| 건너뛰기 링크 | 포커스 시 나타나는 접근성 안내 텍스트, `.skip-link` | 여기에 입력 |
| 헤더 사이트명 | 좌상단 브랜드 텍스트, `.site-brand` | 여기에 입력 |
| 헤더 사이트 설명문 | 브랜드 아래 보조 설명, `.site-brand__desc` | 여기에 입력 |
| 글로벌 네비게이션 링크 | 카테고리 및 about 링크, `.site-nav a` | 여기에 입력 |
| 헤더 검색 액션 | 현재는 글자 없이 아이콘만 있는 버튼, `.site-action` | 해당 없음 |
| 푸터 브랜드 | 푸터의 사이트명, `.site-footer__brand` | 여기에 입력 |
| 푸터 카피 | 푸터의 스택 설명 문구, `.site-footer__copy` | 여기에 입력 |

## 페이지 상단과 공통 액션

이 구간은 홈, 목록, taxonomy, about, search, archive 같은 상단 소개 영역과 여러 화면에서 재사용되는 버튼형 텍스트를 모아둔 부분입니다. 큰 인상과 탐색 리듬을 정할 때 같이 보는 편이 좋습니다.

| 화면 요소 | 설명 | 사용자가 넣을 폰트 |
| --- | --- | --- |
| 페이지 히어로 아이브로우 | `collection`, `search`, `about`, `archive`, `category` 같은 상단 라벨, `.page-hero__eyebrow` | 여기에 입력 |
| 페이지 히어로 제목 | 목록형 페이지의 메인 타이틀, `.page-hero__title` | 여기에 입력 |
| 페이지 히어로 리드문 | 제목 아래 소개 문장, `.page-hero__lede` | 여기에 입력 |
| 공통 버튼 라벨 | 더보기, 홈으로, 아카이브, 검색 같은 버튼 텍스트, `.button` | 여기에 입력 |
| 페이지네이션 텍스트 | 이전, 다음, 페이지 번호, `.pagination a`, `.pagination span` | 여기에 입력 |
| 빈 상태 제목 | 글이 없을 때 보이는 제목, `.empty-state h2` | 여기에 입력 |
| 빈 상태 설명문 | 글이 없을 때 보이는 안내 문장, `.empty-state p` | 여기에 입력 |

## 홈과 목록 카드

홈의 전문 카드, 일반 목록 카드, 태그 목록 카드, 포맷 카드처럼 글을 훑어보는 화면의 텍스트를 한자리에 모았습니다. 같은 카드라도 제목, 메타, 요약, 보조 링크의 성격이 다르므로 분리해서 적는 편이 이후 적용할 때 안전합니다.

| 화면 요소 | 설명 | 사용자가 넣을 폰트 |
| --- | --- | --- |
| 카드 메타 정보 | 날짜, 카테고리, 포맷 배지에 공통으로 쓰이는 메타 텍스트, `.meta`, `.meta__item`, `.badge` | 여기에 입력 |
| 홈 전문 카드 제목 | 홈 스트림의 큰 제목, `.entry-full__title` | 여기에 입력 |
| 홈 전문 카드 보조 링크 | 원문, `¶ 내 글` 링크 문구, `.entry-full__source` | 여기에 입력 |
| 홈 전문 카드 본문 | 홈에서 전문 또는 요약 형태로 보이는 본문, `.entry-full__body`, `.entry-full .prose-body` | 여기에 입력 |
| 목록 카드 제목 | 일반 목록과 태그 목록 카드 제목, `.entry-summary__title` | 여기에 입력 |
| 목록 카드 요약 | 일반 목록과 태그 목록 카드의 스니펫, `.entry-summary__snippet` | 여기에 입력 |
| 목록 카드 보조 링크 | 링크 포맷에서만 보이는 `¶ 내 글`, `.entry-summary__aux` | 여기에 입력 |
| 포맷 카드 메타 정보 | 포맷 뱃지, 날짜, 카테고리, `.format-card__meta` | 여기에 입력 |
| 포맷 카드 제목 | 포맷 페이지의 카드 제목, `.format-card__title` | 여기에 입력 |
| 포맷 카드 요약 | 포맷 페이지의 설명과 발췌, `.format-card__summary` | 여기에 입력 |
| 포맷 카드 보조 링크 | 원문, `¶ 내 글` 링크 문구, `.format-card__links` | 여기에 입력 |

## 아카이브 화면

아카이브는 연도와 월, 날짜와 제목이 동시에 보이는 구조라서 정보 우선순위가 분명해야 합니다. 숫자와 제목을 분리해서 적어두면 나중에 숫자형 메타와 제목형 텍스트를 다르게 조정하기 쉽습니다.

| 화면 요소 | 설명 | 사용자가 넣을 폰트 |
| --- | --- | --- |
| 아카이브 연도 제목 | 연도 구획 제목, `.archive-year__title` | 여기에 입력 |
| 아카이브 월 제목 | 월 구획 제목, `.archive-month__title` | 여기에 입력 |
| 아카이브 날짜 | 각 글의 월일 표기, `.archive-item time` | 여기에 입력 |
| 아카이브 글 제목 | 아카이브 목록의 글 제목 링크, `.archive-item > a` | 여기에 입력 |
| 아카이브 카테고리 메타 | 제목 옆 보조 분류 텍스트, `.archive-item__meta` | 여기에 입력 |
| 아카이브 포맷 배지 | 아카이브 목록의 포맷 표시, `.archive-item .badge` | 여기에 입력 |

## 포스트 상세와 일반 페이지 본문

상세 화면과 about 같은 일반 페이지의 읽기 영역입니다. 여기서는 제목과 메타뿐 아니라 실제 읽는 본문, 소제목, 인용문, 코드, 캡션처럼 서로 다른 읽기 경험을 만드는 텍스트를 분리해 두는 것이 중요합니다.

| 화면 요소 | 설명 | 사용자가 넣을 폰트 |
| --- | --- | --- |
| 포스트 메인 제목 | 상세 페이지 제목, `.post-detail__title` | 여기에 입력 |
| 포스트 메타 정보 | 상세 페이지의 날짜, 카테고리, 포맷 정보, `.post-detail .meta`, `.post-detail .meta__item`, `.post-detail .badge` | 여기에 입력 |
| 일반 본문 문단 | 포스트와 일반 페이지의 기본 문단, `.prose-body p` | 여기에 입력 |
| 일반 본문 목록 | 포스트와 일반 페이지의 리스트 텍스트, `.prose-body li` | 여기에 입력 |
| 본문 소제목 H2 | 본문 내 주요 소제목, `.post-detail .prose-body h2`, `.page-article .prose-body h2` | 여기에 입력 |
| 본문 소제목 H3 | 본문 내 하위 소제목, `.post-detail .prose-body h3`, `.page-article .prose-body h3` | 여기에 입력 |
| 본문 보조 소제목 H4 | 본문 내 더 작은 소제목, `.prose-body h4` | 여기에 입력 |
| 인용문 | 블록 인용문, `.prose-body blockquote` | 여기에 입력 |
| 인라인 코드 | 문장 안 코드, `.prose-body p code`, `.prose-body li code` | 여기에 입력 |
| 코드 블록 | `pre` 와 내부 코드, `.prose-body pre`, `.prose-body pre code` | 여기에 입력 |
| 이미지 캡션 | 그림 설명문, `.prose-body figcaption` | 여기에 입력 |
| 소스 콜아웃 라벨 | 원문 링크 상자 상단 라벨, `.source-callout__label` | 여기에 입력 |
| 소스 콜아웃 URL | 원문 링크 본문, `.source-callout__url` | 여기에 입력 |
| 태그 칩 | 글 하단 태그 버튼, `.chip-list a`, `.tag-list a` | 여기에 입력 |

## 검색과 댓글 패널

검색은 외부 라이브러리 UI 이지만 현재 CSS 에서 텍스트 계층을 직접 제어하고 있으므로 입력 시트에 포함했습니다. 댓글은 사이트가 직접 렌더링하는 패널 헤더까지만 다루고, 실제 위젯 내부 글꼴은 이 시트의 적용 범위 밖에 둡니다.

| 화면 요소 | 설명 | 사용자가 넣을 폰트 |
| --- | --- | --- |
| 검색 추천 라벨 | `추천 키워드` 와 같은 안내 텍스트, `.search-shell__suggestions` | 여기에 입력 |
| 검색 추천 키워드 | 추천 키워드 링크, `.search-shell__suggestions a` | 여기에 입력 |
| 검색 입력창 텍스트 | Pagefind 검색창 입력값과 플레이스홀더, `.pagefind-ui__search-input` | 여기에 입력 |
| 검색 결과 제목 | Pagefind 결과의 제목 링크, `.pagefind-ui__result-title` | 여기에 입력 |
| 검색 결과 발췌문 | Pagefind 결과의 요약 텍스트, `.pagefind-ui__result-excerpt` | 여기에 입력 |
| 검색 메시지와 버튼 | 결과 없음 메시지, 더 보기 버튼 등, `.pagefind-ui__message`, `.pagefind-ui__button` | 여기에 입력 |
| 댓글 패널 제목 | `Discussion` 제목, `.discussion-panel__head h2` | 여기에 입력 |
| 댓글 패널 안내문 | `Powered by commentbox.io` 및 비활성 안내문, `.discussion-panel__head p`, `.muted` | 여기에 입력 |

## 404 화면

404 화면은 일반 페이지와 별도 톤을 쓰고 있어서 숫자와 제목, 설명문을 따로 적는 편이 좋습니다. 큰 숫자를 장식처럼 쓰더라도 실제로는 텍스트이므로 이 시트에 포함합니다.

| 화면 요소 | 설명 | 사용자가 넣을 폰트 |
| --- | --- | --- |
| 404 숫자 | 큰 `404` 숫자, `.error-stage__number` | 여기에 입력 |
| 404 제목 | 오류 화면의 메인 타이틀, `.error-stage__title` | 여기에 입력 |
| 404 설명문 | 오류 안내 문장, `.error-stage__lede` | 여기에 입력 |
| 404 버튼 라벨 | 홈, 아카이브, 검색 이동 버튼, `.error-stage .button` | 여기에 입력 |

## 제외 범위 메모

헤더의 검색 아이콘과 브랜드 마크, 각종 SVG 아이콘, 소셜 임베드 내부 UI, commentbox 위젯 내부 댓글 텍스트는 현재 사이트의 일반적인 폰트 지정 체계로 직접 관리하는 대상이 아니므로 이 문서에서 제외했습니다. 이후 실제 적용 단계에서 폰트 토큰을 만들 때는 이 시트의 항목명만 기준으로 삼고, 여기 없는 요소는 새 항목을 임의로 추가하지 않는 것을 기본 원칙으로 잡습니다.
